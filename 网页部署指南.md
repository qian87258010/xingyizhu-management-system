# 🌐 星奕筑管理系统 - 网页端部署指南

## 📝 概述

这个应用可以部署成网页，通过浏览器访问，无需安装任何软件！

支持的部署平台：
- ✅ Vercel（推荐，最简单）
- ✅ Netlify
- ✅ GitHub Pages
- ✅ 自己的服务器

---

## 🚀 方式一：Vercel 部署（推荐，最简单）

### 特点
- ✅ 完全免费
- ✅ 自动 HTTPS
- ✅ 全球 CDN 加速
- ✅ 自动部署（推送代码自动更新）
- ✅ 3分钟完成部署

### 步骤

#### 1. 准备代码（本地）

```bash
# 确保项目可以正常构建
npm install
npm run build
```

#### 2. 上传到 GitHub（如果还没有）

```bash
# 初始化 git（如果还没有）
git init
git add .
git commit -m "初始提交"

# 在 GitHub 上创建一个新仓库，然后：
git remote add origin https://github.com/你的用户名/星奕筑管理系统.git
git branch -M main
git push -u origin main
```

#### 3. 部署到 Vercel

**方式 A：通过网页界面（推荐）**

1. 访问 [vercel.com](https://vercel.com)
2. 点击 "Sign Up" 使用 GitHub 账号登录
3. 点击 "New Project"
4. 选择你的 GitHub 仓库
5. 配置如下：
   - Framework Preset: **Vite**
   - Build Command: `npm run build`
   - Output Directory: `dist`
6. 点击 "Deploy"
7. 等待 1-2 分钟，完成！

**方式 B：使用命令行**

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录
vercel login

# 3. 部署
vercel

# 4. 按提示操作
# 5. 部署到生产环境
vercel --prod
```

#### 4. 获取网址

部署完成后，你会得到一个网址，例如：
```
https://星奕筑管理系统.vercel.app
```

🎉 完成！现在任何人都可以通过这个网址访问你的系统！

## 📄 方式三：GitHub Pages（免费）

### 步骤

#### 1. 创建部署脚本

```bash
chmod +x scripts/deploy-github-pages.sh
./scripts/deploy-github-pages.sh
```

#### 2. 手动部署

```bash
# 1. 构建
npm run build

# 2. 进入构建目录
cd dist

# 3. 初始化 git
git init
git add -A
git commit -m 'deploy'

# 4. 推送到 gh-pages 分支
git push -f https://github.com/你的用户名/仓库名.git main:gh-pages

# 5. 回到项目根目录
cd -
```

#### 3. 启用 GitHub Pages

1. 访问 GitHub 仓库
2. Settings → Pages
3. Source: 选择 `gh-pages` 分支
4. 点击 Save

网址：`https://你的用户名.github.io/仓库名/`

## 🖥️ 方式四：自己的服务器

### 使用 Nginx

#### 1. 构建项目

```bash
npm run build
```

#### 2. 上传到服务器

```bash
# 使用 SCP 上传
scp -r dist/* user@your-server.com:/var/www/html/
```

#### 3. 配置 Nginx

创建 `/etc/nginx/sites-available/xingyizhu`：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
}
```

#### 4. 启用配置

```bash
sudo ln -s /etc/nginx/sites-available/xingyizhu /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 使用 Apache

创建 `.htaccess` 在 dist 目录：

```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>
```

## 🐛 常见问题

### Q1: 刷新页面后 404
**A:** 需要配置服务器路由。已在各配置文件中包含。

### Q2: 样式丢失
**A:** 检查 `vite.config.ts` 中的 `base` 配置

### Q3: 部署后白屏
**A:** 
1. 检查浏览器控制台错误
2. 确认构建成功：`npm run build`
3. 本地测试：`npx vite preview`

### Q4: 环境变量不生效
**A:** 
1. 确保变量名以 `VITE_` 开头
2. 重新构建项目
3. 在代码中使用 `import.meta.env.VITE_XXX`
