## 🎯 推荐流程

### 新手（第一次使用）

1️⃣ **本地测试**
```bash
npm install
npm run dev
```
访问 http://localhost:5173，熟悉系统功能

2️⃣ **部署上线**
```bash
./quick-start.sh
# 选择 "3) 部署到 Vercel"
```
得到网址，分享给团队

3️⃣ **开始使用** 🎉

---

### 开发者（需要修改）

1️⃣ **本地开发**
```bash
npm run dev
```

2️⃣ **修改代码**
- 查看 `/src/app/` 目录
- 修改组件和样式

3️⃣ **测试**
```bash
npm run build
npx vite preview
```

4️⃣ **部署**
```bash
git push
# Vercel 自动部署
```

---

### 团队（内网使用）

1️⃣ **Docker 部署**
```bash
docker-compose up -d
```

2️⃣ **访问**
```
http://服务器IP
```

3️⃣ **配置反向代理**（可选）
- Nginx
- 自定义域名

## 💡 温馨提示

### ✅ 推荐做法
- 先在本地测试功能
- 确认无误后再部署
- 定期备份重要数据
- 使用 Git 管理代码

### ⚠️ 注意事项
- 数据存储在 LocalStorage
- 刷新页面不会丢失数据
- 不同浏览器数据独立
- 清除浏览器缓存会清除数据

### 🔒 安全建议
- 修改默认测试账号密码
- 不要在公网存储敏感数据
- 定期导出重要数据
- 使用 HTTPS 访问
