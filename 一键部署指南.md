## 📋 部署前准备（5分钟）

在 Figma Make 中，点击右上角"下载"按钮，将项目下载到您的电脑。

### 2️⃣ 上传到 GitHub

**方式一：使用 GitHub Desktop（推荐，最简单）**

1. 下载安装 [GitHub Desktop](https://desktop.github.com/)
2. 登录您的 GitHub 账号
3. 点击 `File` → `Add Local Repository`
4. 选择项目文件夹
5. 点击 `Publish repository`
6. 完成！

**方式二：使用命令行**

```bash
# 1. 进入项目文件夹
cd 星奕筑管理系统

# 2. 初始化 Git（如果还没有）
git init

# 3. 添加所有文件
git add .

# 4. 提交
git commit -m "Initial commit - 星奕筑管理系统"

# 5. 在 GitHub 创建新仓库后，关联并推送
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

## 🌐 方案一：部署到 Vercel（推荐）

### ⚡️ 为什么选择 Vercel？
- ✅ 完全免费
- ✅ 3分钟部署完成
- ✅ 自动 HTTPS 安全加密
- ✅ 全球 CDN 加速
- ✅ 每次更新自动部署

### 📝 部署步骤

#### 1. 访问 Vercel
打开 https://vercel.com

#### 2. 登录
点击 `Sign Up` 或 `Log In`，使用 GitHub 账号登录

#### 3. 导入项目
1. 点击 `Add New...` → `Project`
2. 选择您的 GitHub 仓库 `星奕筑管理系统`
3. 点击 `Import`

#### 4. 配置（使用默认设置）
- Framework Preset: **Vite** （自动检测）
- Build Command: `npm run build`
- Output Directory: `dist`
- ✅ 保持默认设置即可

#### 5. 部署
点击 `Deploy` 按钮，等待 2-3 分钟

#### 6. 完成！🎉
部署成功后，您会得到类似这样的网址：
```
https://xingyizhu.vercel.app
```

### 🔗 获取您的网址

部署完成后，在 Vercel 控制台可以看到：
- **Production URL**: 您的正式网址（分享给团队使用）
- **Preview URLs**: 预览网址（用于测试）

### 📱 自定义域名（可选）

如果您有自己的域名（如 `www.yourcompany.com`）：

1. 在 Vercel 项目页面点击 `Settings` → `Domains`
2. 输入您的域名
3. 按照提示在域名服务商处添加 DNS 记录
4. 完成！

## 🎯 方案二：部署到 Netlify

### 📝 部署步骤

#### 1. 访问 Netlify
打开 https://netlify.com

#### 2. 登录
使用 GitHub 账号登录

#### 3. 导入项目
1. 点击 `Add new site` → `Import an existing project`
2. 选择 `Deploy with GitHub`
3. 授权 Netlify 访问您的 GitHub
4. 选择仓库 `星奕筑管理系统`

#### 4. 配置（使用默认设置）
- Build command: `npm run build`
- Publish directory: `dist`
- ✅ Netlify 会自动读取 `netlify.toml` 配置

#### 5. 部署
点击 `Deploy site` 按钮

#### 6. 完成！🎉
您会得到类似这样的网址：
```
https://xingyizhu.netlify.app
```

### 🔗 自定义网站名称

1. 在 Netlify 控制台点击 `Site settings`
2. 点击 `Change site name`
3. 输入您想要的名字（如 `xingyizhu-management`）
4. 网址变成：`https://xingyizhu-management.netlify.app`

## 🖥️ 方案三：部署到 GitHub Pages

### 📝 部署步骤

#### 1. 运行部署脚本

```bash
# 进入项目文件夹
cd 星奕筑管理系统

# 赋予执行权限
chmod +x scripts/deploy-github-pages.sh

# 执行部署
./scripts/deploy-github-pages.sh
```

#### 2. 启用 GitHub Pages

1. 访问您的 GitHub 仓库
2. 点击 `Settings` → `Pages`
3. Source 选择 `gh-pages` 分支
4. 点击 `Save`

#### 3. 完成！🎉
您的网址：
```
https://你的用户名.github.io/仓库名
```

## ❓ 常见问题

### Q1: 部署后页面是空白的？
**A:** 检查浏览器控制台，通常是路由配置问题。
- Vercel/Netlify 已自动配置好
- GitHub Pages 需要修改 `vite.config.ts` 中的 base 路径

### Q2: 登录后刷新页面会退出？
**A:** 这是正常的，因为使用的是前端模拟数据。如需持久化登录，请连接 Supabase 数据库。

### Q3: 可以分享给其他人使用吗？
**A:** 可以！直接分享网址，其他人打开即可使用。
- 但数据是各自浏览器本地保存的
- 如需多人共享数据，建议连接 Supabase

### Q4: 网址可以改成中文吗？
**A:** 不能，域名只能使用英文、数字和连字符。
但您可以：
- 购买自己的域名（如 `xingyizhu.com`）
- 在 Vercel/Netlify 中绑定自定义域名

### Q5: 部署是免费的吗？
**A:** 是的！
- Vercel 免费版：每月 100GB 流量
- Netlify 免费版：每月 100GB 流量
- GitHub Pages 完全免费

### Q6: 数据会丢失吗？
**A:** 当前版本数据存储在浏览器中：
- 清除浏览器缓存会丢失
- 换电脑/浏览器无法同步
- 建议重要数据定期导出备份
